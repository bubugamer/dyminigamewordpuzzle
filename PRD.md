

# **1. 产品目标**

- 提供一款轻量级脑洞类解谜小游戏。
    
- 通过“图 + 谐音”制造反差感、惊喜感。
    
- 每局耗时 3–10 秒，满足碎片化场景。
    
- 首版仅验证玩法，不做复杂系统。
    

---

# **2. 用户场景**

  

**典型路径：**

  

看到一张“提示区 + 谜面区”的组合图

→ 解读提示含义

→ 从谜面场景联想到谐音梗

→ 在输入框填写成语或短语

→ 提交 → 判断正确/错误 → 下一关。

---

# **3. 核心玩法逻辑**

- 展示一张组合图（上半提示，下半谜面）。
    
- 用户输入推测的成语或短语。
    
- 完整比对答案集合，正确进入下一关。
    

  

## **3.1 关卡结构**

- **image**：单张组合图（你提供的素材样式）。
    
    - 上半部分：提示图 + 文字提示（文字为图片的一部分）。
        
    - 下半部分：谜面图 + “这是____”等文字（同样为图片素材本身）。
        
    
- **answers**：标准答案数组。
    
- **altAnswers**：等价谐音/变体答案数组。
    

  

## **3.2 玩家任务**

  

观察组合图 → 根据谐音推理出最终答案。

  

## **3.3 判定规则**

- 提交后进行全字比对。
    
- 支持容错：
    
    - 繁简互通
        
    - 去除标点与空格（全角/半角）
        
    - 支持同义与谐音变体（altAnswers）。
        
    

---

# **4. 游戏流程（MVP）**

  

## **4.1 主要流程**

1. 首页
    
2. 开始游戏
    
3. 展示关卡组合图
    
4. 输入答案
    
5. 提交
    
6. 正确 → 过关提示 → 下一关
    
7. 错误 → “再想想”
    
8. 最后一关通关 → 通关页
    

  

## **4.2 流程示意**

```
首页 → 开始
          ↓
     第 N 关
          ↓
   输入答案 → 提交
      ↓        ↓
   正确     错误
      ↓        ↓
    下一关    重试
```

---

# **5. 界面结构**

  

## **5.1 首页**

- 游戏名
    
- “开始游戏”按钮
    

  

## **5.2 关卡页**

- 顶部：单张组合图（原图比例展示，自适应宽度）
    
- 下方：
    
    - 答案输入框
        
    - “提交”按钮
        
    - 错误提示（轻量）
        
    

  

## **5.3 过关反馈**

- 浮层：“正确！”
    
- 按钮：“下一关”
    

  

## **5.4 通关页**

- “恭喜通关”
    
- 按钮：“重新开始”
    

---

# **6. 关卡管理**

  

## **6.1 数据结构**

```
{
  "id": 1,
  "image": "xxx.png",
  "answers": ["八方来财"],
  "altAnswers": ["八方来柴"]
}
```

## **6.2 匹配清洗规则**

- 去除空格（全角/半角）
    
- 去除标点
    
- 繁简统一（转简体）
    
- 再进行字符串匹配
    

---

# **7. 交互逻辑**

  

## **7.1 输入**

- 中文输入
    
- 提交按钮有 0.2s loading
    
- 正确：跳关
    
- 错误：轻度抖动
    

  

## **7.2 动画**

- 正确：绿色缩放动画
    
- 错误：抖动
    
- 全部 ≤ 300ms
    

---

# **8. 技术要求（MVP）**

- H5 / 抖音小游戏框架
    
- 本地 JSON 存储关卡
    
- 无需服务端
    
- 首版提供 7 关
